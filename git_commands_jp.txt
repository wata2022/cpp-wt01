
# Gitでよく使うコマンド

## リポジトリのセットアップ

- `git init`: 現在のディレクトリに新しいGitリポジトリを作成します。

- `git clone [リポジトリのURL]`: リモートにあるリポジトリをローカルにコピー（クローン）します。

## 変更の記録

- `git status`: リポジトリの状態（変更されたファイル、ステージングエリアにあるファイルなど）を表示します。

- `git add [ファイル名]`: ファイルの変更を次のコミットのためにステージングエリアに追加します。
  - `git add .` または `git add -A`: カレントディレクトリ以下の全ての変更をステージングします。

- `git commit -m "[コミットメッセージ]"`: ステージングエリアにある変更を、メッセージを付けてローカルリポジトリに記録（コミット）します。
  - `git commit -am "[コミットメッセージ]"`: ファイルの変更のステージングとコミットを同時に行います（追跡中のファイルのみ）。

- `git diff`: ファイルの変更差分を表示します。
  - `git diff --staged`: ステージングエリアと前回のコミットとの差分を表示します。

## リモートとの連携

- `git push [リモート名] [ブランチ名]`: ローカルリポジトリのコミットをリモートリポジトリに送信（プッシュ）します。
  - 例: `git push origin main`

- `git pull [リモート名] [ブランチ名]`: リモートリポジトリの最新の変更を取得し、現在のブランチにマージします。
  - 例: `git pull origin main`

- `git fetch [リモート名]`: リモートリポジトリの最新情報を取得します（マージはしません）。

- `git remote -v`: 登録されているリモートリポジトリの情報を表示します。

## ブランチ操作

- `git branch`: ブランチの一覧を表示します。現在のブランチには `*` が付きます。

- `git branch [新しいブランチ名]`: 新しいブランチを作成します。

- `git checkout [ブランチ名]`: 指定したブランチに切り替えます。
  - `git checkout -b [新しいブランチ名]`: 新しいブランチを作成して、そのブランチにすぐに切り替えます。

- `git merge [ブランチ名]`: 指定したブランチの変更を、現在のブランチに統合（マージ）します。

- `git branch -d [ブランチ名]`: マージ済みのブランチを削除します。
  - `git branch -D [ブランチ名]`: マージされていないブランチを強制的に削除します。

## 履歴の確認

- `git log`: コミットの履歴を表示します。
  - `git log --oneline`: 履歴を1行で簡潔に表示します。
  - `git log --graph`: ブランチのマージ履歴などをグラフィカルに表示します。

## 変更の取り消し

- `git reset [ファイル名]`: `git add` したファイルを取り消し、ステージングエリアから戻します。

- `git checkout -- [ファイル名]`: ファイルの変更を、前回のコミットの状態まで戻します。

- `git reset --hard [コミットID]`: 指定したコミットの状態までリポジトリを完全に戻します（注意：以降のコミットは失われます）。
